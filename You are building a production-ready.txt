You are building a production-ready Roman Academy academic dashboard and student portal for a coaching institute that runs 11th, 12th, CET, and limited add-on Commerce/NEET support.

IMPORTANT GOAL
Build a lightweight, elite, student-focused website with minimal database load, minimal charts, fast UX, and clear workflow. Do not create heavy or noisy visuals. Use one main graph on the student profile, a few compact subject bars, and expandable sections for detailed analytics. Keep the database lean and store only what is necessary for analysis and workflow.

TECH STACK
- Next.js App Router
- TypeScript
- Tailwind CSS
- shadcn/ui
- Framer Motion for small transitions only
- Recharts for graphs
- Prisma or a similarly simple ORM
- PostgreSQL for production, SQLite acceptable for local/dev
- WhatsApp integration through a provider abstraction
- AI provider abstraction with primary + fallback model support

DESIGN DIRECTION
- Clean, professional, elite academic UI
- Dark navy + white + gold accent theme consistent across all pages
- Minimal clutter
- Rounded cards, subtle shadows, no excessive animations
- Make it feel like a premium coaching platform, not a generic school app
- One strong profile card, one main progress graph, subject-wise compact bars, and simple tabs/sections
- Use popups for first-time walkthrough and important alerts

PRODUCT SUMMARY
Roman Academy is a coaching-class dashboard for:
- 11th batch starting July 2026
- Existing 12th batch already running from June 2026
- CET preparation for PCM
- Board support through English, practicals, CS/IT, and extra passing subjects
- Optional Commerce and NEET add-on modules
- Personal attention at class level, but not a tiny private batch
- Teacher/admin-led mark entry and syllabus updates
- Student login with password
- AI-generated analysis and WhatsApp draft messages
- Rank-style leaderboard like a gaming or coding platform

HARD CONSTRAINTS
- Keep the app lite
- Do not store question-by-question answer data
- Store only marks, chapter names, subject totals, test type, date, rank, and teacher notes
- No unnecessary graph overload
- No duplicated data
- Each update should sync only the relevant student and relevant batch
- Late joiners should not be forced into old completed curriculum; earlier topics should be shown as catch-up or revision only
- For 12th students attending 11th CET lectures, track that separately as a second learning stream
- Every admin action should be editable
- Teacher must review AI-generated WhatsApp drafts before sending

CORE ROLES
1. Teacher/Admin
2. Student

- Student username = name-based username

- After login, route to the correct role dashboard

AI / AUTOMATION RULES
- Use AI only for analysis, summaries, weak-topic detection, and WhatsApp message drafting
- AI should generate:
  - concise subject summary
  - weak chapters/topics
  - next action plan
  - parent/student WhatsApp draft
  - monthly and weekly progress summary
- Always keep teacher review before sending WhatsApp
- Add a fallback AI provider if the primary fails
- Add a fallback messaging path if WhatsApp API fails
- Use a prompt-to-structured-update parser on the admin page:
  - teacher writes a natural-language command
  - system converts it to a fixed structured update preview
  - teacher confirms before saving
- Important alerts should appear as popups on the website
- Quarterly test alerts should appear inside the student login/profile area
- Weekly and monthly summaries can be sent by WhatsApp after teacher approval

- Do not overload months
